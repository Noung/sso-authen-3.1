System Architecture

The SSO Authentication System provides centralized authentication services using the OpenID Connect standard:

Core Components:
  • OIDC Provider Integration: Connect with PSU SSO, Google, Microsoft, and custom providers
  • JWT Token Management: Secure token generation and validation with configurable expiration
  • Multi-Client Support: Manage multiple applications with individual configurations
  • Admin Panel: Web-based management interface with comprehensive APIs
  • Audit & Analytics: Usage statistics, activity logs, and backup capabilities

Quick Start Guide

Get your application connected to the SSO system in minutes

1. Register Your Application

Contact your system administrator to register your application and obtain:

  • client_id - Your unique application identifier
  • client_secret - Secret key for API authentication (if using JWT mode)
  • redirect_uri - Callback URL for your application

2. Implement Login Flow

Redirect users to the SSO login endpoint:

// Redirect to SSO login
function loginWithSSO() {
    const params = new URLSearchParams({
        client_id: 'your_client_id',
        redirect_uri: 'http://yourapp.com/callback'
    });
    
    window.location.href = `https://sso.yourorg.com/public/login.php?${params}`;
}
<?php
// PHP redirect to SSO
$params = http_build_query([
    'client_id' => 'your_client_id',
    'redirect_uri' => 'http://yourapp.com/callback.php'
]);

header("Location: https://sso.yourorg.com/public/login.php?{$params}");
exit;
?>
// React login component
import React from 'react';

const LoginButton = () => {
    const handleLogin = () => {
        const params = new URLSearchParams({
            client_id: 'your_client_id',
            redirect_uri: window.location.origin + '/callback'
        });
        
        window.location.href = `https://sso.yourorg.com/public/login.php?${params}`;
    };
    
    return (
        <button onClick={handleLogin} className="btn btn-primary">
            Login with SSO
        </button>
    );
};

3. Handle Callback

Process the authentication response in your callback handler:

// Handle callback with JWT token
const urlParams = new URLSearchParams(window.location.search);
const token = urlParams.get('token');

if (token) {
    // Store token securely
    localStorage.setItem('jwt_token', token);
    
    // Decode payload (optional - for display only)
    const payload = JSON.parse(atob(token.split('.')[1]));
    console.log('User:', payload.data);
    
    // Redirect to dashboard
    window.location.href = '/dashboard';
} else {
    console.error('Authentication failed');
}
<?php
// Handle callback in PHP
session_start();

if (isset($_GET['token'])) {
    $jwt_token = $_GET['token'];
    
    // Store in session
    $_SESSION['jwt_token'] = $jwt_token;
    $_SESSION['logged_in'] = true;
    
    // Decode payload for user info
    $payload = json_decode(base64_decode(str_replace('_', '/', str_replace('-', '+', explode('.', $jwt_token)[1]))), true);
    $_SESSION['user_data'] = $payload['data'];
    
    // Redirect to dashboard
    header('Location: /dashboard.php');
    exit;
} else {
    echo 'Authentication failed';
}
?>

Authentication Overview

Understanding the SSO authentication mechanisms

Two Authentication Levels

The SSO system provides two distinct authentication layers:

User Authentication (OIDC)

Purpose: Authenticate end users through client applications

Method: OpenID Connect flow with JWT tokens

Usage: Public-facing applications, user login

  • No API key required
  • Stateless JWT tokens
  • Provider integration
  • Automatic session management
Admin Authentication

Purpose: Access admin panel and management APIs

Method: Session-based authentication

Usage: System administration, API management

  • Session validation required
  • Protected API endpoints
  • Audit logging
  • Role-based access

JWT Token Structure

JWT tokens returned by the SSO system contain the following structure: 

{
  "iss": "sso-authen-service",
  "iat": 1640995200,
  "exp": 1641081600,
  "data": {
    "id": "user123",
    "username": "john.doe",
    "name": "John Doe", 
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com",
    "picture": "https://example.com/avatar.jpg",
    "department": "IT Department"
  }
}
Token Validation: Always verify the JWT signature using the provided JWT secret key. The token expiration is configurable (default: 24 hours).

Admin Panel Authentication

Securing access to administrative APIs

Important: All admin panel APIs require session-based authentication. Ensure your requests include valid session cookies.

Authentication Flow

To access admin panel APIs, you must first authenticate through the admin login endpoint:

POST /admin/login.php
Authentication Required
Request Parameters
Parameter Type Required Description
username string Required Admin username
password string Required Admin password
remember_me boolean Optional Extend session duration (default: false)
Example Request
curl -X POST "https://sso.yourorg.com/admin/login.php" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=securepassword&remember_me=true" \
  -c cookies.txt
Response
{
  "success": true,
  "message": "Login successful",
  "user": {
    "id": 1,
    "username": "admin",
    "name": "System Administrator",
    "role": "super_admin",
    "last_login": "2023-01-01 10:30:00"
  }
}

Session Validation

After successful login, all subsequent requests to admin APIs must include the session cookies. The system automatically validates session authenticity and permissions.

GET /admin/api/validate-session.php
Authentication Required

Validate current session status and retrieve user information.

Example Request
curl -X GET "https://sso.yourorg.com/admin/api/validate-session.php" \
  -b cookies.txt
Response
{

    "username": "admin",
    "name": "System Administrator",
    "role": "super_admin",
    "permissions": ["manage_clients", "view_stats", "backup_restore"]
  }
}

Python

Example of implementing SSO authentication in a Python application. 

# Redirect to SSO login
import webbrowser

def login_with_sso():
    params = {
        'client_id': 'your_client_id',
        'redirect_uri': 'http://yourapp.com/callback'
    }
    url = f"https://sso.yourorg.com/public/login.php?{urllib.parse.urlencode(params)}"
    webbrowser.open(url)

# Handle callback with JWT token
import jwt

def handle_callback(token):
    if token:
        # Store token securely
        with open('jwt_token.txt', 'w') as f:
            f.write(token)
        
        # Decode payload (optional - for display only)
        payload = jwt.decode(token, options={"verify_signature": False})
        print('User:', payload['data'])
        
        # Redirect to dashboard
        print('Redirecting to dashboard...')
    else:
        print('Authentication failed')

Java

Example of implementing SSO authentication in a Java application.

// Redirect to SSO login
import java.net.URI;
import java.net.URISyntaxException;

public class SSOAuth {
    public static void loginWithSSO() {
        String clientId = "your_client_id";
        String redirectUri = "http://yourapp.com/callback";
        String url = String.format("https://sso.yourorg.com/public/login.php?client_id=%s&redirect_uri=%s", clientId, redirectUri);
        try {
            java.awt.Desktop.getDesktop().browse(new URI(url));
        } catch (URISyntaxException | IOException e) {
            e.printStackTrace();
        }
    }

    // Handle callback with JWT token
    public static void handleCallback(String token) {
        if (token != null) {
            // Store token securely
            try (FileWriter writer = new FileWriter("jwt_token.txt")) {
                writer.write(token);
            } catch (IOException e) {
                e.printStackTrace();
            }
            
            // Decode payload (optional - for display only)
            try {
                Map<String, Object> payload = JWT.decode(token).getPayload();
                System.out.println("User: " + payload.get("data"));
                
                // Redirect to dashboard
                System.out.println("Redirecting to dashboard...");
            } catch (JWTDecodeException e) {
                e.printStackTrace();
            }
        } else {
            System.out.println("Authentication failed");
        }
    }
}

Troubleshooting

Common issues and solutions

Note: If you encounter any issues, refer to the following troubleshooting steps.

Authentication Errors

Issue: Authentication fails with an error message.

Solution: Verify that your credentials are correct and that the redirect URI matches the one registered in the SSO system.

Token Expiration

Issue: JWT token expires too quickly.

Solution: Adjust the token expiration setting in the SSO admin panel.

Network Issues

Issue: Unable to connect to the SSO server.

Solution: Check your network connection and ensure that the SSO server is reachable.

FAQ

Frequently Asked Questions

Note: If you have any questions, feel free to ask.

What is SSO?

SSO stands for Single Sign-On, a method of access control that enables users to authenticate once and gain access to multiple applications without re-entering their credentials.

How do I register my application?

Contact your system administrator to register your application and obtain the necessary credentials.

Can I use SSO with multiple providers?

Yes, SSO supports integration with multiple providers such as PSU SSO, Google, Microsoft, and custom providers.

What is JWT?

JWT stands for JSON Web Token, a compact, URL-safe means of representing claims to be transferred between two parties.
